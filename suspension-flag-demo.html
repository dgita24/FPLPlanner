🚩 Suspension Flag Logic - Visual Demo

Problem Statement

Players suspended for specific gameweeks (e.g., GW22-23) were showing red flags in future gameweeks (GW24+) when they should be available.

Solution Overview

The fix introduces gameweek-aware flag logic that:

  • Parses suspension duration from player news text
  • Calculates the suspension end gameweek
  • Only shows flags during the suspension period
  • Removes flags when viewing gameweeks after suspension expires

Implementation Details

Helper Function: getSuspensionEndGW()

function getSuspensionEndGW(player, currentGW) { if (!player.news) return null; // Parse: "Suspended for 2 matches" or "Suspended for 2 games" const matchesMatch = player.news.match(/Suspended for (\d+) (?:match|matches|game|games)/i); if (matchesMatch) { const suspendedMatches = parseInt(matchesMatch[1]); return currentGW + suspendedMatches - 1; } // Parse: "Unavailable until gameweek 24" const gwMatch = player.news.match(/(?:until|in) gameweek (\d+)/i); if (gwMatch) { return parseInt(gwMatch[1]) - 1; } return null; // Injury or unknown - treat as indefinite }

Helper Function: isSuspensionExpiredForGW()

function isSuspensionExpiredForGW(player, gwToCheck, currentGW) { // Only applies to suspended players if (player.status !== 's') return false; const suspensionEndGW = getSuspensionEndGW(player, currentGW); // If we can't parse suspension end, treat as indefinite if (!suspensionEndGW) return false; // Suspension has expired if viewing GW is after the suspension end GW return gwToCheck > suspensionEndGW; }
