🐛 GW22 Flag Bug Fix - chance_of_playing Issue

The Bug

User Report: After the FPL semantics fix, Keane still showed NO flag in GW22.

❌ What Was Happening

Keane's data in GW22:

  • status: 's' (suspended)
  • news: "Suspended until 31 January"
  • chance_of_playing_this_round: 100 ⚠️

The problem: The code saw chance_of_playing_this_round = 100 and hid the flag, even though Keane was suspended for GW22!

Root Cause

// OLD (BUGGY) - Checked chance_of_playing for ALL players function shouldShowPlayerFlag(player, viewingGW, currentGW, events) { if (!player.status || player.status === 'a') return false; // This applied to EVERYONE, including suspended players! if (viewingGW === currentGW && player.chance_of_playing_this_round === 100) { return false; // Hide flag - WRONG for suspensions! } // Suspension check came AFTER, so never reached if (player.status === 's' && isSuspensionExpired(...)) { return false; } return true; }

Why this caused the bug:

  1. Keane is suspended for GW22, but FPL API sets chance_of_playing_this_round = 100
  2. This field means "100% chance of playing in FUTURE gameweeks" not "can play THIS gameweek"
  3. Old code: "chance is 100%? Hide flag!" ❌
  4. Result: No flag shown in GW22 even though he's suspended

The Fix

// NEW (CORRECT) - Only check chance_of_playing for NON-suspended players function shouldShowPlayerFlag(player, viewingGW, currentGW, events) { if (!player.status || player.status === 'a') return false; // NEW: Only for injuries/doubtful, NOT suspensions! if (player.status !== 's') { if (viewingGW === currentGW && player.chance_of_playing_this_round === 100) { return false; // OK to hide for injuries } } // For suspensions, ALWAYS use our calculated end date if (player.status === 's' && isSuspensionExpired(...)) { return false; } return true; }

✅ Key Insight

Suspensions have definite end dates that we can calculate accurately from the news text or match count.

Injuries/doubtful status are uncertain, so chance_of_playing fields provide useful additional information.

Solution: Use chance_of_playing only for injuries/doubtful, not for suspensions.
